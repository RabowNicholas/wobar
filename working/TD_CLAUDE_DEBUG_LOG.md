## Recurring Themes

### audio
Failures related to audio pipeline construction, operator selection, and signal analysis.
- `filterCHOP` ≠ frequency filter — always use `audiofilterCHOP`
- HP filter output is bipolar — always use `rmspower` in `analyzeCHOP`, never `average`
- Analysis CHOPs (null_audio, band outputs) are not audio streams — they run at cook rate, not 44100 Hz

### export
Failures related to recording and render output setup.
- `null_audio` is an analysis CHOP at rate 30, not an audio waveform — use the source `audiofileinCHOP` for `moviefileoutTOP` audiochop
- **Non-commercial TD: no h264/h265 export** → use `mjpa` (.mov) or PNG image-sequence; audio `pcm16` (mp3/aac can be license-gated)

### feedback
- `feedbackTOP` defaults to **128×128** — set the loop ops' res explicitly, then pulse `par.resetpulse` (it caches old res). **CANONICAL wiring (reconciled 2026-06-10, supersedes "wire target to input[0]"):** `input[0]` = fresh content upstream of the loop; `par.top` = the loop target. Target-on-input[0] creates the circular reference behind the cook-dependency-loop warning, which is NOT cosmetic — it can block cooking. First-frame "Not enough sources" clears on cook. Full pattern: WOBAR_TD_AGENT_RULES.md → Feedback Chain Rules.

### twozero
- exec_server (`td_execute_python`): list-comps / genexprs / lambdas can't reference outer-scope vars bound in the same block — `NameError`. Use plain `for` loops with `.append()`.

### expressions
- `sin`/`cos`/`pi` need the `math.` prefix (`math.sin()` etc.) — bare `sin` is a `NameError`. **RULE (recurred 2026-06-09).**
- A par-expression error is not local: the erroring op re-cooks every frame and can force expensive **upstream** re-cooks (e.g. a 4K HDRI re-decode → fps crash). When fps tanks unexpectedly, check `op('/x').errors(recurse=True)` first.

### other
- Transparent / instanced (POPX) glass can't produce a usable depth pass — alpha-blended geometry doesn't write reliable depth, so depth-buffer DOF is out.
- Animated environment-light IBL is expensive: `envlightmapprefilter='automatic'` ≈ 142 ms/frame. Fix: prefilter `off` + downsample the env map (resTOP ~512px) → ~0 ms.
